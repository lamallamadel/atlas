    ### Design Tokens
    - **Colors**: Primary, secondary, semantic colors, and accessibility compliant palettes
    - **Typography**: Modular scale, font weights, line heights
    - **Spacing**: 4px grid system with consistent spacing tokens
    - **Shadows**: 5-level elevation system for depth and hierarchy

    ### Guidelines
    - **Accessibility**: WCAG AA compliance, keyboard navigation, screen readers
    - **Best Practices**: Component usage patterns and anti-patterns
    - **Do and Don't**: Visual examples of correct and incorrect usage

    ### Components
    - **Buttons**: Primary, secondary, text, icon buttons
    - **Forms**: Inputs, selects, checkboxes, validation
    - **Cards**: Property cards, stats cards, interactive cards
    - **Badges**: Status badges, count badges, semantic badges
    - **Illustrations**: Empty states, error states, success states

    - **Code Snippets**: Copy-ready HTML, CSS, and TypeScript code
    - **Static Export**: Generate static documentation site for team sharing

  <div className="sb-section">
    ## üé® Design Philosophy

    Our design system is built on these principles:

    ### Consistency
    - Unified visual language across all touchpoints
    - Predictable component behavior and interactions
    - Systematic use of colors, spacing, and typography

    ### Accessibility
    - WCAG AA compliant color contrast ratios (4.5:1 minimum)
    - Keyboard navigation support for all interactive elements
    - Screen reader optimized markup and ARIA labels
    - Focus indicators with 2px solid outlines

    ### Modularity
    - Composable components that work together seamlessly
    - Flexible design tokens for easy theming
    - Reusable patterns and templates

    ### Performance
    - Optimized bundle sizes with lazy loading
    - Efficient CSS with design tokens
    - Minimal JavaScript for interactions
  </div>

    ## üõ†Ô∏è Building Blocks

    ### Design Tokens (CSS Custom Properties)
    ```css
    --color-primary-500: #2c5aa0;
    --spacing-4: 1rem; /* 16px */
    --shadow-2: 0 4px 6px -1px rgba(0, 0, 0, 0.1);
    --radius-xl: 0.75rem; /* 12px */
    ```

    ### Component Usage
    ```html
    <button class="btn btn--primary">
      Primary Action
    </button>
    ```

    ### Angular Components
    ```typescript
    import { BadgeComponent } from '@app/components/badge';
    ```
  </div>

  <div className="sb-section">
    ## üì± Responsive Design

    Our components are built mobile-first with responsive breakpoints:
    - **Mobile**: 0-575px
    - **Tablet**: 576-991px
    - **Desktop**: 992-1199px
    - **Wide**: 1200px+

    Use the viewport toolbar to test responsive behavior.
  </div>

  <div className="sb-section">
    ## üåô Dark Mode

    All components support dark mode out of the box. Toggle the background in the toolbar to test dark mode appearance.

    Dark mode is implemented using CSS custom properties and the `.dark-theme` class:

    ```css
    .dark-theme {
      --color-background: #1a1a1a;
      --color-text: #ffffff;
    }
    ```
