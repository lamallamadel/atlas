## What's in This Storybook?

### üé® Design Tokens
Complete token system for consistent design:
- **Colors**: Primary, secondary, semantic, and neutral palettes
- **Typography**: Modular scale, font weights, line heights
- **Spacing**: 4px grid system with 14 spacing levels
- **Shadows**: 6 elevation levels for depth
- **Border Radius**: 9 radius values from subtle to full
- **Animations**: Duration scale and easing functions

### üß© Components
50+ documented components including:
- **Buttons**: 6 variants (primary, secondary, outlined, text, icon, FAB)
- **Forms**: Inputs, selects, textareas, checkboxes, radio buttons with validation
- **Cards**: Multiple layouts for different content types
- **Badges**: Status indicators and labels with semantic colors
- **Dialogs**: Modals, confirmations, form dialogs
- **Charts**: Line, bar, pie/doughnut with Chart.js
- **Icons**: Real estate icons and Material Icons
- **Loading States**: Spinners, skeleton loaders, progress bars
- **Empty States**: No data, no results, error states
- **Illustrations**: Lottie animations for engaging UX

### üìö Guidelines
Best practices and guidelines for:
- **Accessibility**: WCAG AA compliance checklist
- **Usage Patterns**: Do's and don'ts for each component
- **Code Examples**: TypeScript, HTML, and CSS snippets
- **Keyboard Navigation**: Keyboard shortcut documentation
- **Responsive Design**: Breakpoint strategy
- **Dark Theme**: Theme implementation guidelines

## Technology Stack

- **Angular 16.2**: Modern Angular framework
- **Angular Material**: Material Design components
- **TypeScript**: Type-safe development
- **Chart.js**: Data visualization
- **Lottie**: Animation library
- **Storybook 7.6**: Component documentation
- **SCSS/CSS**: Styling with CSS custom properties

## Accessibility Standards

All components meet:
- WCAG 2.1 Level AA
- 4.5:1 minimum contrast ratio for text
- 3:1 minimum contrast ratio for UI components
- Keyboard navigation support
- Screen reader compatibility
- Focus indicators (2px solid outline)
- Touch target size (44√ó44px minimum)

## Getting Started

1. **Browse Components**: Navigate using the sidebar
2. **View Stories**: Each component has multiple story variations
3. **Interactive Controls**: Use the Controls tab to modify props
4. **Check Accessibility**: View the A11y tab for WCAG compliance
5. **Copy Code**: Find code snippets in each story
6. **Test Responsiveness**: Use viewport addon to test different sizes
7. **Toggle Theme**: Switch between light and dark themes

## Contributing

This design system is actively maintained. To contribute:

1. Follow existing patterns and conventions
2. Ensure WCAG AA accessibility compliance
3. Test in both light and dark themes
4. Provide comprehensive documentation
5. Include code examples
6. Add accessibility guidelines
